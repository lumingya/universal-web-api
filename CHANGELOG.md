# 更新日志

## 2.7.5 - 2026-04-03
- 更新教程与文档口径：教程正文暂不展开“独立 Cookie 标签页”高级功能，并在教程 FAQ / README 已知问题中明确补充当前限制。
- 补充已知问题说明：当前不建议依赖“同一受控浏览器窗口内的独立 Cookie”方案；在浏览器原生菜单等场景下，独立 Cookie 会话仍可能出现稳定性问题，默认仍推荐共享 Cookie 路线。
- 同步刷新版本号与教程资源版本戳，避免前端缓存继续显示旧教程内容。

## 2.7.1 - 2026-03-28
- 修复命令引擎在高优先级命令插入工作流后容易遗留 `command_interrupt` 状态的问题；手动执行或命令内嵌工作流不再因为陈旧中断标记而在启动即返回 `cancelled`。
- 改进 Arena 预设重命名链路：重命名站点预设时，现在会同步更新 `commands.json` 里的 `preset_name` 引用以及当前活动标签页上的预设引用；同时保留不带 `预设_` 前缀旧名字的别名兼容，避免历史命令配置失效。
- 修复非流式工作流收尾时偶发的 `generator ignored GeneratorExit` 噪音，并增强隐身输入前的焦点恢复逻辑；输入框焦点临时丢失时会先补一次静默校验与重新激活，减少 `clipboard_focus_failed`。
- 修复命令 `run_js` 在未显式写顶层 `return` 的 IIFE 脚本里经常拿到 `None` 的问题；现在命令动作会自动补一次 `return` 重试，避免把原本成功的 probe 误判成失败并反复重置 `page_check` 锁存。
- 补强 Arena 命令自动化：新增九宫格/人机验证清 Cookie 与 Cookie 弹窗自动接受命令，细分 Cloudflare 与 reCAPTCHA 触发条件，并把 `page_check` 快照扩展为可识别 reCAPTCHA 外层 iframe/selector 标记，减少不同验证页之间的误触发串扰。
- 调整 Arena side-by-side 左侧流错误处理：`lmarena_side_left` 解析到左侧目标流明确报错时，现在会直接终止当前工作流而不是回退整页 DOM；同时浏览器层会对这种左侧终止错误自动静默重试一次，避免右侧报错误伤左侧成功流，也减少左侧短暂失败造成的人工重发。
- 为 `page_check` 增加详细命中日志：触发时除了命令名外，还会记录命中的关键词列表和页面快照预览，便于直接从运行日志中定位到底是哪段文本或标记触发了命令。
