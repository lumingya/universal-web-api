# 更新日志

## 2.7.0 - 2026-03-26
- 将插件市场切换为公共模式优先：市场目录默认优先读取 GitHub 公共索引，并提供 GitHub 仓库与目录入口；投稿默认改为打开 GitHub 公共投稿页，提交时会先复制完整 JSON 预览，避免大体积站点配置或命令包因为 URL 长度限制而丢失内容；同时补充仓库内的市场投稿 Issue 模板，并将 `config/marketplace.json` 调整为公共市场清单格式，保留远程失败时回退本地市场的能力。
- 新增独立“插件市场”页面：控制台右上角入口现在直接跳转到单独的市场页，不再把市场内容挤在原有配置/日志/命令区域里；同时将市场首页重做为更紧凑的列表式布局，移除占用大面积空间的头图区与示例展示块，默认按下载量排序，并保留站点配置/命令系统的筛选、预览、投稿与导入流程；站点配置继续支持“覆盖导入”和“另存为新预设”，命令系统也可直接作为命令包导入；另外修复了主控制台“插件市场”入口按钮的乱码显示，并清空了本地市场示例数据，避免空市场时混入演示项。
- 修复工作流步骤“是否必要”勾选语义与实际执行逻辑相反的问题：控制面板与可视化工作流编辑器现统一改为“必需步骤”语义，勾选后找不到元素才会报错，未勾选时会按可选步骤跳过，避免界面显示与 `optional` 字段行为不一致。
- 重构调试日志基础结构：统一日志格式中明确保留时间、请求上下文和模块名，并为前端日志收集器补充 `seq`、`level`、`kind`、`logger`、`request_id`、`message_text`、`display_message` 等结构化字段，便于后续筛选、追踪和定位问题。
- 新增高频 DEBUG 限频机制：对于轮询、等待和重复命中的调试日志，统一支持按 key 节流输出，并在下一次输出时附带 `suppressed=N` 摘要，显著减少无意义刷屏。
- 收敛网络监听、文本输入、发送重试、命令触发检查、标签页等待等高频链路的 debug 内容：从“每一步都打印”调整为“关键状态变化 + 阶段摘要 + 失败上下文”，在降低噪音的同时保留绝大多数排障所需信息。
- 调整 tool calling 与消息解析相关日志：请求消息改为摘要化输出，响应解析日志补充长度、模式、工具名和预览，移除多层重复打印的大段原文，避免日志页被长文本淹没。
- 优化控制面板日志页与日志 API：日志轮询从基于时间戳切换为基于递增序号的 `after_seq` 游标，降低边界漏日志风险；前端日志页新增模块名和请求 ID 展示，正文优先显示干净消息文本，提升阅读和定位效率。
- 清理启动阶段的无条件调试输出：将直接 `print` 的 Python 路径改为受日志级别控制的 debug 日志，避免非调试模式下也产生无用输出。
- 新增可选的日志可爱化开关，并补充教程说明：`INFO` 与主要 `DEBUG` 日志现在都支持统一润色为更口语化、更容易读懂的文案，并保留原始日志文本用于排障对照；其中长文本分块、网络监听、输入校验、文件粘贴、隐身发送、执行器、标签页池等关键 debug 路径都已补充“小鹿”风格提示，可在控制面板的“设置 → 浏览器常量 → 日志”中单独开关。
- 修复控制面板“选择器测试”弹窗的状态同步与重复测试体验：从选择器列表点“测试”后，弹窗会正确带入当前选择器、超时时间和“高亮显示”选项；在弹窗里修改或勾选高亮后，现在可以直接连续重复点击“测试”，无需先关闭再重新打开。
- 修复选择器测试“找到元素但浏览器里没有真正高亮”的问题：后端高亮逻辑改为直接作用在已命中的真实元素上，并在需要时尝试激活对应受控浏览器标签页，减少带特殊字符的选择器或重复测试时高亮失效的情况。

## 2.6.9 - 2026-03-26
- 将文件粘贴、图片粘贴和发送前等待统一改为“附件状态驱动”逻辑：不再主要依赖固定等待秒数，而是综合观察输入区 DOM 变化、附件/预览节点、处理中标记以及发送按钮忙闲状态，再决定附件是否真正挂载成功、何时允许发送。
- 修复 Gemini 长文本文件粘贴容易出现“文件已上传但发送前又白等 20 秒”的问题；发送前门禁不再重复要求重新观测一轮附件，只要当前状态稳定且发送按钮恢复可提交，就会继续发送。
- 修复文件粘贴在“页面已有附件活动但未能完全确认”时错误降级到普通文本输入的问题；现在会停止回退，避免出现“文件和文本被重复一起发送”的错误行为。
- 图片粘贴链路同步接入同一套附件状态监视器，不再仅靠每张图片粘贴后的短暂固定等待来猜测上传是否成功。
- 更新教程页“文件粘贴 / File Attach”说明：补充新版附件状态检测与发送门禁的工作方式，并同步修正默认启用文件粘贴的内置站点列表。

## 2.6.6 - 2026-03-25
- 改进 `start.bat` 的 Python 依赖安装流程：当默认 PyPI 源安装 `requirements.txt` 失败时，会自动回退到国内镜像重试，并支持通过 `PIP_MIRROR_URL` 自定义镜像地址，尽量减少用户手动干预。
- 将标签页池的“卡死强制释放超时”加入控制面板的“设置 -> 浏览器常量 -> 标签页池”，保存后会写入根目录的 `config/browser_config.json`，并在运行中的标签页池已初始化时同步更新生效。
- 修复控制面板保存 `TAB_POOL_*` 参数时回退到旧值的问题；现在“最大标签页数 / 最小保留标签页数 / 空闲超时 / 占用等待超时 / 卡死强制释放超时”都能正确保存并回显。
- 调整标签页池的页面筛选逻辑：初始化扫描、增量扫描、健康检查和域名刷新现在都会排除本地链接与浏览器内部页，包括 `localhost`、`127.0.0.1`、局域网/内网 IP、`about:`、`chrome://`、`devtools://` 等地址，避免这些页面被纳入标签页池解析。
- 改写教程页“快速开始”，补充了新手首次启动时如何区分受控浏览器与普通浏览器、如何在受控浏览器中打开 Gemini 并登录，以及如何复用项目内 `chrome_profile/` 或自定义 `BROWSER_PROFILE_DIR` 的登录态。
- 调整教程页已适配站点卡片的交互方式：点击站点卡片时不再直接在当前教程浏览器跳转，而是复制目标网址并提示用户去受控浏览器中打开，降低误操作概率。
- 修复函数调用兼容层在多轮工具调用场景下的上下文组织问题：工具返回结果不再被注入为 `system` 指令，而是作为普通对话上下文传回模型，并补充“收到 Tool Result 后可继续调用工具或结束回答”的提示，改善连续调用续轮的稳定性。
- 增强函数调用响应解析的容错能力：现在会尽量修复 JSON 字符串中混入的真实换行、回车、Tab、控制字符以及命令参数里的 Windows 路径转义问题，提升脏格式 `tool_calls` 输出的识别成功率。
- 将全局站点默认配置与本地默认预设覆盖分离，使 `sites.local.json` 不再污染 `sites.json`，同时在通用配置保存后仍能保留本地覆盖项。
- 新增完整的设置备份导入/导出功能，覆盖 env 配置、浏览器常量、update-preserve 设置、选择器定义以及本地仪表盘偏好。
- 在前端设置页加载、编辑、重置和保存时，保留 `tab_pool` 等嵌套浏览器常量以及命令的周期检查设置。
- 新增基于域名的标签页路由，支持 `/url/{domain}/v1/...` 端点，并包含 `gemini.google.com -> gemini.com` 别名支持；同时继续保留固定的 `/tab/{index}/v1/...` 路由作为第二种策略。
- 更新标签页池 UI，展示实时的 `current_domain`、`domain_url`、域名路由端点和固定标签页路由端点，并为两类端点分别提供复制操作。
- 修复命令引擎的域名解析和预设执行行为，包括安全的“跟随站点默认预设”支持，以及修正 `click_element` 的站点配置查找逻辑。
- 移除过时的提取器前端入口点，并从仪表盘中删除未被引用的提取器 UI 文件。
- 改进请求校验错误信息，在请求被拒绝时同时报告当前消息数量/长度与配置的限制值。
- 修复 stealth 工作流的延迟暴涨问题：根据真实墙钟时间为 CDP 鼠标移动和空闲漂移进行节奏控制，并限制事件密度；在保持常规网络和 DOM 流程正常的同时，将一次可复现的 Gemini stealth 运行从约 233 秒降至约 27 秒。
- 修复 `start.bat` 的浏览器启动流程，使专用 Chrome 配置目录能够被可靠启动和检测。
- 在 `start.bat` 中强制使用 UTF-8 Python 输出，避免不同启动模式下 `patch_drissionpage.py` 出现编码失败。
- 将 `start.bat` 统一为一致的 `CRLF` 行尾，防止混合换行导致 `cmd` 解析错误。
- 已验证在 `BROWSER_PROFILE_DIR=C:\Users\QIU\AppData\Local\UniversalWebApiProfile` 下可端到端启动成功，包括 `9222` 上的 Chrome DevTools 和 `8199` 上的服务健康检查。
