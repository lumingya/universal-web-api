# 更新日志

## 2.7.1 - 2026-03-28
- 修复命令引擎在高优先级命令插入工作流后容易遗留 `command_interrupt` 状态的问题；手动执行或命令内嵌工作流不再因为陈旧中断标记而在启动即返回 `cancelled`。
- 改进 Arena 预设重命名链路：重命名站点预设时，现在会同步更新 `commands.json` 里的 `preset_name` 引用以及当前活动标签页上的预设引用；同时保留不带 `预设_` 前缀旧名字的别名兼容，避免历史命令配置失效。
- 修复非流式工作流收尾时偶发的 `generator ignored GeneratorExit` 噪音，并增强隐身输入前的焦点恢复逻辑；输入框焦点临时丢失时会先补一次静默校验与重新激活，减少 `clipboard_focus_failed`。
- 修复命令 `run_js` 在未显式写顶层 `return` 的 IIFE 脚本里经常拿到 `None` 的问题；现在命令动作会自动补一次 `return` 重试，避免把原本成功的 probe 误判成失败并反复重置 `page_check` 锁存。
- 补强 Arena 命令自动化：新增九宫格/人机验证清 Cookie 与 Cookie 弹窗自动接受命令，细分 Cloudflare 与 reCAPTCHA 触发条件，并把 `page_check` 快照扩展为可识别 reCAPTCHA 外层 iframe/selector 标记，减少不同验证页之间的误触发串扰。
- 调整 Arena side-by-side 左侧流错误处理：`lmarena_side_left` 解析到左侧目标流明确报错时，现在会直接终止当前工作流而不是回退整页 DOM；同时浏览器层会对这种左侧终止错误自动静默重试一次，避免右侧报错误伤左侧成功流，也减少左侧短暂失败造成的人工重发。
- 为 `page_check` 增加详细命中日志：触发时除了命令名外，还会记录命中的关键词列表和页面快照预览，便于直接从运行日志中定位到底是哪段文本或标记触发了命令。

## 2.7.0 - 2026-03-26
- 修复公共市场项目卡片里作者信息经常只显示“本地投稿/社区贡献”而看不到真实来源的问题：待审核投稿和新收录项目现在都会优先补齐 GitHub 提交人；如果投稿里填写了自定义作者名，同时还会保留单独的 GitHub 提交者标识，方便区分展示作者与实际提交账号。
- 为公共市场补上“作者本人 / 管理员可撤回或下架”的权限链路：GitHub 已连接但没有仓库维护权限的投稿作者，现在也可以撤回自己尚未审核的投稿，或下架自己已经收录到公共市场的项目；仓库管理员则继续保留通过、拒绝和下架能力。
- 加固公共市场索引的 GitHub 读取链路：远程 `marketplace.json` 现在会先尝试 `raw.githubusercontent.com`，遇到 TLS EOF、连接重置、超时等临时网络错误时会先做一次轻量重试；如果 raw 源仍失败，并且当前地址本来就指向仓库内的标准市场索引，则会自动切到 GitHub Contents API 读取同一份文件，尽量避免页面直接掉到“只剩本地缓存”的状态。
- 为公共 GitHub 市场补上“维护权限即管理员”的审核链路：市场页新增本地保存的 GitHub 审核入口，可用当前浏览器里的 GitHub token 校验对公共仓库的维护权限；只有校验通过后，待审核投稿卡片上才会显示“通过 / 拒绝”按钮。后端同时新增 GitHub 权限检测、远程 `config/marketplace.json` 写回、issue 关闭以及待审核与已收录条目的去重逻辑，避免审核后因 issue 尚未关闭而在列表里出现重复项。
- 继续精简插件市场底栏入口：移除“查看目录 JSON”，底部只保留 GitHub 仓库和公共投稿页两个外链入口。
- 继续精简插件市场底栏：移除“公共插件市场 / 项数 / 待审核 / 累计下载 / 默认排序 / GitHub 公共索引”等底部状态标签，只保留目录、仓库和投稿入口，减少低价值信息占位。
- 调整插件市场底部信息区位置：将“公共插件市场 / 项数 / 待审核 / GitHub 入口”从顶部工具区移到页面最底部，改成独立页脚栏，避免顶部控制区信息过密，同时让整体布局更接近常见插件市场的浏览结构。
- 继续收紧插件市场布局：重新压缩页面版心、头部和筛选工具区的高度，把项目列表改成更接近横向信息流的紧凑卡片，缩小统计块与操作按钮占位，减少浅色模式下大片留白和“整块白板”式的空洞感。
- 改进公共市场的 GitHub 限额与离线兜底策略：待审核投稿读取现在优先走 GitHub API，命中匿名 `403` 限额后会自动切换到公开 issue 页面抓取；同时支持通过环境变量提供 GitHub token 提升 API 配额，并把成功读取到的公共市场结果落盘缓存到本地文件，之后即使远端索引或待审核读取失败，也能优先回退到本地缓存继续显示和导入。
- 继续优化独立“插件市场”页面的列表布局与深浅色主题：将市场来源/统计信息收拢到底部信息带，压缩单个项目卡片的右侧操作区与统计块，减少大面积空白；同时重做浅色模式的背景、面板和按钮对比度，避免出现顶部区域和卡片区域发灰、发闷且层次不清的问题。
- 修复公共市场“投稿后列表里完全看不见”的问题：市场页现在会把 GitHub 上尚未收录进正式目录、但标题符合“[市场投稿] ...”格式的 open issue 也识别为“待审核”项目并展示在列表里；同时保留只按市场专用标题/标记/标签识别的边界，避免普通 bug issue 仅因正文里带 JSON 就被误判为市场投稿。
- 调整公共市场投稿的 GitHub 引导文案与剪贴板内容：现在 GitHub 页面会自动填好基本信息与简介，应用只复制 JSON 代码块，避免用户把整段内容重复粘贴后出现“基本信息/简介”双份的问题。
- 将插件市场切换为公共模式优先：市场目录默认优先读取 GitHub 公共索引，并提供 GitHub 仓库与目录入口；投稿默认改为打开 GitHub 公共投稿页，提交时会先复制完整 JSON 预览，避免大体积站点配置或命令包因为 URL 长度限制而丢失内容；同时补充仓库内的市场投稿 Issue 模板，并将 `config/marketplace.json` 调整为公共市场清单格式，保留远程失败时回退本地市场的能力。
- 新增独立“插件市场”页面：控制台右上角入口现在直接跳转到单独的市场页，不再把市场内容挤在原有配置/日志/命令区域里；同时将市场首页重做为更紧凑的列表式布局，移除占用大面积空间的头图区与示例展示块，默认按下载量排序，并保留站点配置/命令系统的筛选、预览、投稿与导入流程；站点配置继续支持“覆盖导入”和“另存为新预设”，命令系统也可直接作为命令包导入；另外修复了主控制台“插件市场”入口按钮的乱码显示，并清空了本地市场示例数据，避免空市场时混入演示项。
- 修复工作流步骤“是否必要”勾选语义与实际执行逻辑相反的问题：控制面板与可视化工作流编辑器现统一改为“必需步骤”语义，勾选后找不到元素才会报错，未勾选时会按可选步骤跳过，避免界面显示与 `optional` 字段行为不一致。
- 重构调试日志基础结构：统一日志格式中明确保留时间、请求上下文和模块名，并为前端日志收集器补充 `seq`、`level`、`kind`、`logger`、`request_id`、`message_text`、`display_message` 等结构化字段，便于后续筛选、追踪和定位问题。
- 新增高频 DEBUG 限频机制：对于轮询、等待和重复命中的调试日志，统一支持按 key 节流输出，并在下一次输出时附带 `suppressed=N` 摘要，显著减少无意义刷屏。
- 收敛网络监听、文本输入、发送重试、命令触发检查、标签页等待等高频链路的 debug 内容：从“每一步都打印”调整为“关键状态变化 + 阶段摘要 + 失败上下文”，在降低噪音的同时保留绝大多数排障所需信息。
- 调整 tool calling 与消息解析相关日志：请求消息改为摘要化输出，响应解析日志补充长度、模式、工具名和预览，移除多层重复打印的大段原文，避免日志页被长文本淹没。
- 优化控制面板日志页与日志 API：日志轮询从基于时间戳切换为基于递增序号的 `after_seq` 游标，降低边界漏日志风险；前端日志页新增模块名和请求 ID 展示，正文优先显示干净消息文本，提升阅读和定位效率。
- 清理启动阶段的无条件调试输出：将直接 `print` 的 Python 路径改为受日志级别控制的 debug 日志，避免非调试模式下也产生无用输出。
- 新增可选的日志可爱化开关，并补充教程说明：`INFO` 与主要 `DEBUG` 日志现在都支持统一润色为更口语化、更容易读懂的文案，并保留原始日志文本用于排障对照；其中长文本分块、网络监听、输入校验、文件粘贴、隐身发送、执行器、标签页池等关键 debug 路径都已补充“小鹿”风格提示，可在控制面板的“设置 → 浏览器常量 → 日志”中单独开关。
- 修复控制面板“选择器测试”弹窗的状态同步与重复测试体验：从选择器列表点“测试”后，弹窗会正确带入当前选择器、超时时间和“高亮显示”选项；在弹窗里修改或勾选高亮后，现在可以直接连续重复点击“测试”，无需先关闭再重新打开。
- 修复选择器测试“找到元素但浏览器里没有真正高亮”的问题：后端高亮逻辑改为直接作用在已命中的真实元素上，并在需要时尝试激活对应受控浏览器标签页，减少带特殊字符的选择器或重复测试时高亮失效的情况。

## 2.6.9 - 2026-03-26
