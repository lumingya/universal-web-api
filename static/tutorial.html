⚠️ 作者的话

1️⃣ 函数调用目前问题很多

Function Calling 这块目前还有不少问题，极其依赖 AI 本身的理解能力。换句话说，它不是一个已经完全收敛、可以闭眼依赖的稳定功能，用的时候请降低预期，多做实测。

2️⃣ 作者主要维护 Gemini，其它站点不保证及时跟进

作者本人平时主要使用的是 Gemini 站点，因此其它站点并不能保证及时维护。出了问题更推荐加群反馈，不建议发 Issue，也不建议去 Discord 反馈，因为作者基本不看。

3️⃣ 有问题优先先问 AI

很多问题其实不需要等作者处理，尤其是选择器失效、配置写错、工作流顺序不对这类问题。优先把现象、日志、元素树交给 AI 帮你分析，通常会比直接等回复更快。

4️⃣ 站点不能用时，优先怀疑选择器；其次才是解析器

一般来说，一个站点突然不能用了，不是解析器出问题了，就是选择器出问题了，而更常见的是选择器失效。

选择器更新其实很简单：

  1. 在页面上对想点击或想读取的元素右键，选择“检查”。
  2. 把对应的元素树截图或复制出来。
  3. 把这段元素树发给 AI，让 AI 告诉你新的选择器应该怎么写。

解析器问题会麻烦一些：如果是解析器出问题，通常需要拦截网络请求、看响应结构，再做逆向分析。这比改选择器麻烦得多。

能自己更新就尽量自己更新：如果你已经修好了，欢迎把更新后的配置文件或代码文件发给作者，作者会非常感谢你。

5️⃣ 反馈 Bug 请描述清楚，并尽量附上 DEBUG 日志

如果要反馈 Bug，请尽量把问题描述清楚一点，并提供 DEBUG 级别日志。如果你觉得自己表达不清楚，先让 AI 帮你整理一下语言再发，因为作者有时候确实听不太懂对方到底想表达什么。

6️⃣ 教程虽然覆盖很全，但细节不一定写透

这个教程文件已经尽量把主要功能都覆盖了，但很多细节没有写到特别展开。你如果有能力，也欢迎一起补充和完善文档。

7️⃣ 很多功能本质上是为酒馆 / RP 场景服务的

作者本人主要玩的还是酒馆之类的 RP 场景，因此很多功能本来就是为这个目标服务的。换句话说，除此之外的一些功能，维护上基本属于“放养”状态，请理解这一点。

🆕 新增站点

新增站点有两条路线：让系统自动识别，或者手动新增并自己配置。建议第一次先走自动识别，跑通后再人工修正。

路线 A：AI 自动识别

  1. 进入 控制面板 → 设置 → 环境配置，填好 HELPER_API_KEY、HELPER_BASE_URL、HELPER_MODEL。
  2. 在脚本控制的浏览器里打开目标站点，并停留在真正可聊天的页面。
  3. 对该站点发起第一次真实 API 请求。
  4. 如果这个域名还不在 config/sites.json 中，后端会自动抓取 HTML 并调用辅助 AI 做页面分析。
  5. 分析成功后，会自动生成主预设并写入站点配置文件。

关键点：自动识别不是点“新增站点”按钮触发，而是未知域名第一次实际请求时触发。也就是说，只打开页面不发请求，后端不会自动写入配置。

路线 B：手动新增并自己配置

  1. 点击左下角 新增站点。
  2. 输入域名，例如 chat.example.com。
  3. 进入“主预设”，先补三个最小核心选择器。
  4. 再补最短工作流并逐个测试。
  5. 最后保存并实际调用一次 API。

🔌 连接 API

本项目提供兼容 OpenAI 格式 的接口。

⚙️ 客户端通用配置参数：

  • 服务提供商 (Provider)：选择 OpenAI、OpenAI 兼容 或 自定义。
  • API 接口地址 (Base URL)：
    • 默认（自动分配标签页）：http://127.0.0.1:8199/v1
    • 指定站点域名：http://127.0.0.1:8199/url/gemini.com/v1（自动匹配该站点的标签页）
    • 指定标签页：http://127.0.0.1:8199/tab/1/v1（编号见标签页池）
    • 部分软件需完整路径：http://127.0.0.1:8199/v1/chat/completions
  • API 密钥 (API Key)：任意填写即可（例如：sk-any）。
  • 模型名称 (Model)：取决于你访问的网站。你访问什么网站，实际就是什么模型，所以该项可随意填写。

✅ 测试步骤：当你按照第一步打开了站点并登录之后，填写了对应的 URL 和服务商内容，就可以直接进行测试了。

💡 关于登录状态

  • 推荐登录：建议在网页上登录你的账号，这样 API 也就拥有了你的账号权限（如 Plus 会员、历史记录等）。
  • 免登录：如果该网站在未登录状态下允许对话，则无需登录即可直接通过 API 调用。

⚠️ 关于 SillyTavern (酒馆): 在酒馆中使用 API 测试功能可能会出现 bug，建议直接发送对话进行测试，而非点击测试按钮。

⚠️ 上下文长度限制：受限于网站输入框一次性所能容纳的字符上限，上下文不能太长，否则请求会直接失败。请根据实际情况控制对话长度。对于长文本场景，可以开启「文件粘贴」功能来绕过部分限制。

已测试站点单次最大发送长度：

  • ChatGPT：约 200k。
  • Gemini 官网：无会员约 30k，Pro 会员尚未发现上限。
  • Arena AI：120k。

🧰 函数调用兼容说明

这是新增功能区域。项目现在兼容 OpenAI 标准函数调用格式，可以接收客户端传来的 tools / tool_choice，同时也兼容旧字段 functions / function_call。

工作方式：客户端还是按标准 OpenAI 方式发请求；后端会把工具定义、工具历史和调用约束整理成网页端模型更容易理解的提示内容；模型返回后，后端再尝试把结构化结果解析回 tool_calls。

边界要看清：这不是原生 Tool Calling。它本质上仍然依赖网页端提示词和模型自己的理解能力，不是官方 API 层直接执行的函数调用，所以整体稳定性通常不如原生接口。

什么时候更容易成功

  • 模型本身指令遵循能力强，愿意严格按格式输出
  • 工具数量不多，名称和职责区分清楚
  • 参数结构比较简单，没有太多深层嵌套对象
  • 系统提示词比较干净，没有塞入过多无关要求

什么时候更容易失败

  • 模型智力不够，无法稳定理解“必须输出结构化工具调用”这件事
  • 函数名相似、参数很多、schema 太复杂
  • 提示词太长，模型注意力被正文内容冲散
  • 模型输出里混入解释文字、自然语言总结或格式不完整的 JSON，导致后端无法解析

最重要的预期管理：如果你遇到“命令写错了”“参数对不上”“明明该调工具却直接说了一段话”“后端提示无法解析 tool_calls”这类现象，通常不是你的客户端接错了，而是网页端模型没能稳定完成这次函数调用输出。

实战建议

  1. 先只给 1 到 3 个工具，不要一上来塞十几个
  2. 函数名尽量短、清晰、不要相互撞名
  3. 参数 schema 尽量扁平，先避免深层对象和复杂数组
  4. 优先选择更强、更听话的模型；模型不够强时，规则写再多也可能失败
  5. 如果发现总是解析失败，先简化工具和参数，再考虑换模型

🗂️ 标签页池

标签页池是本项目的核心调度机制。脚本会自动扫描浏览器中打开的所有 AI 网站标签页，并为每个标签页分配一个持久编号。

工作原理

  1. 你在浏览器中打开一个或多个 AI 网站标签页（如 ChatGPT、Gemini 等）
  2. 脚本自动检测并将它们加入标签页池，分配编号（1, 2, 3...）
  3. API 请求到达时，自动选择一个空闲的标签页来处理
  4. 处理完成后，标签页被释放回池中，等待下一个请求

两种路由方式

路由地址格式说明
默认路由 /v1/chat/completions 自动选择一个空闲标签页处理请求
指定站点域名 /url/{域名}/v1/chat/completions 自动匹配该域名对应的标签页，例如 /url/gemini.com/v1/chat/completions
指定标签页 /tab/{编号}/v1/chat/completions 使用特定编号的标签页，如果该标签页忙碌则排队等待

💡 何时使用指定标签页：

  • 当你想让某个客户端始终使用特定的 AI 站点时
  • 当你为不同标签页配置了不同的预设时（见下方「预设系统」）
  • 当你需要确保对话的连续性时

在控制面板中管理

在控制面板中点击「标签页」选项卡，可以：

  • 查看所有标签页的实时状态（空闲/忙碌/错误）
  • 查看每个标签页的当前 URL 和请求计数
  • 复制指定标签页的 API 端点地址
  • 为标签页切换不同的预设配置

⚠️ 注意事项：

  • 标签页编号在脚本运行期间保持不变，重启后会重新分配
  • 关闭浏览器中的标签页后，对应的编号会自动释放
  • 新打开的标签页会在几秒内被自动检测到并加入池中
  • chrome://newtab 等空白页不会被加入池，页面加载完成后才会被识别

🎛️ 预设系统

预设系统允许你为同一个站点创建多套独立的配置方案，并将不同预设分配给不同的标签页。

典型应用场景

标签页预设名称配置差异
标签页 #1 Pro 模型对话 完整工作流（含 new_chat_btn）、较长超时、启用深度提取
标签页 #2 快速识图 简化工作流、启用图片提取、较短超时
标签页 #3 代码助手 启用文件粘贴、高阈值、启用网络监听模式

如何使用预设

1. 创建预设

  1. 在控制面板中选择一个站点
  2. 在「配置」选项卡顶部可以看到预设选择器
  3. 点击「＋ 新建预设」，输入名称后创建
  4. 新预设会克隆当前选中预设的所有配置
  5. 然后可以独立修改新预设的选择器、工作流、提取器等配置
  6. 如需让该预设成为默认，点击「⭐ 设为默认」

2. 为标签页分配预设

  1. 切换到「标签页」选项卡
  2. 在对应标签页的行中找到「预设」下拉框
  3. 选择想要使用的预设即可

3. 通过 API 使用站点域名路由或指定标签页

# 假设当前已有一个 Gemini 站点标签页，标签页 #2 使用"快速识图"预设

# 使用 Gemini 站点标签页
curl http://127.0.0.1:8199/url/gemini.com/v1/chat/completions

# 使用固定编号的快速识图标签页
curl http://127.0.0.1:8199/tab/2/v1/chat/completions

💡 提示：每个预设包含独立的选择器、工作流、流式配置、图片提取、文件粘贴等全套配置。修改一个预设不会影响其他预设。未手动指定时会自动使用站点默认预设（default_preset）。

预设配置结构

{
  "gemini.google.com": {
    "default_preset": "快速识图",
    "presets": {
      "主预设": {
        "selectors": { ... },
        "workflow": [ ... ],
        "stream_config": { ... },
        "image_extraction": { "enabled": false },
        "file_paste": { "threshold": 50000 }
      },
      "快速识图": {
        "selectors": { ... },
        "workflow": [ ... ],
        "image_extraction": { "enabled": true, "mode": "first" },
        "file_paste": { "threshold": 10000 }
      }
    }
  }
}

⚠️ 注意：删除预设不可撤销，且至少需要保留一个预设。如果只有一个预设，删除按钮会被禁用。

🔍 选择器配置 (Selectors)

每个站点（的每个预设）都需要定义一组 CSS 选择器，告诉程序去哪里输入、哪里点击。

"selectors": {
  "input_box": "textarea[id='prompt']",   // 输入框
  "send_btn": "button.send-btn",          // 发送按钮
  "result_container": ".markdown-body",   // AI 回复所在的容器
  "new_chat_btn": "button.new-chat",      // 新建对话按钮
  "temp_chat_btn": "button.temp-chat"     // 临时对话按钮 (自定义)
}
  • input_box: 必填。聊天输入框。
  • send_btn: 必填。发送按钮。
  • result_container: 必填。包含 AI 回复内容的容器。
  • new_chat_btn: 选填。每次新会话开始时点击的按钮。

💡 自定义选择器：你可以自己新增选择器，比如新增一个「临时对话按钮」，然后在工作流里添加点击这个按钮的步骤，程序就会在执行时多点一步临时对话。

🧪 测试选择器：在控制面板的选择器配置项中点击「测试」按钮可以验证选择器是否生效，也可以在「工作流可视化」中查看配置是否正确。

🌊 响应检测配置

定义如何判断 AI 正在输出以及何时结束。支持两种检测模式：

模式是否流式说明
DOM 模式（默认） ✅ 流式 轮询检测页面 DOM 变化，实时输出内容。适用于绝大多数场景
网络拦截模式 ✅ 流式（按站点适配） 拦截浏览器网络请求并解析增量响应，通常更快、更稳

DOM 模式（默认推荐）

DOM 模式通过轮询页面内容变化来检测 AI 的输出，支持流式返回。对不同网站的兼容性有所差异，普遍存在代码块读取不完整的问题（因为代码块的 HTML 渲染结构较复杂）。如果你主要用于角色扮演等纯文本场景，DOM 模式完全够用；如果用于写代码，可能需要配合网络拦截模式。

检测逻辑

生成结束判定 = (连续稳定次数 ≥ 稳定判定次数) AND (静默时间 > 静默超时阈值)

工作原理：

  1. 脚本每隔「默认检查间隔」秒检查一次页面内容
  2. 如果内容有变化，重置计数器
  3. 如果内容没变化，稳定次数 +1，静默时间累加
  4. 当两个条件同时满足时，判定 AI 生成完成
  5. 如果超过「初始等待」时间仍未检测到内容，判定失败

调优建议

场景静默超时阈值稳定判定次数初始等待
快速模型 (GPT-4o)3-5 秒3-5 次60 秒
慢速思考 (o1/Claude)10-15 秒8-12 次300 秒
代码生成8-10 秒6-8 次180 秒
长文生成12-15 秒10 次300 秒

网络拦截模式

⚠️ 重要提醒：网络拦截模式目前仅针对部分站点做了适配。如果某个站点默认没有开启网络拦截，请不要手动开启——大概率是没有适配，或者开启后会产生副作用（如增加被 Cloudflare 检测的风险）。

网络拦截模式通过拦截浏览器的 XHR/Fetch 请求来获取 AI 的响应，并通过站点解析器提取增量内容。对于已适配站点，通常可实现稳定流式输出，且对复杂内容（如代码块）兼容性更好。

💡 启用方式：在控制面板的站点配置中，找到「流式配置」面板，将模式切换为「网络拦截」即可。

网络拦截配置项

配置项说明示例
URL 匹配模式 网络请求 URL 中包含此字符串时拦截（必填） GenerateContent
响应解析器 用于解析拦截到的响应数据的解析器 ID（必填） 根据站点自动匹配
首次响应超时 等待首次有效网络响应的最长时间；超时后会根据模式回退或中止 300 秒
静默超时 拦截到响应后，多久没有新数据判定为完成 3 秒
轮询间隔 检查是否有新的网络响应的频率 0.5 秒

💡 Arena 解析器说明：arena.ai 目前内置了两个响应解析器。lmarena 用于 direct 模式，适合单栏直接对话；lmarena_side_left 用于 side by side 模式，会自动解析左侧内容（即左边模型的输出）。通常不需要手动填写，按站点默认配置即可；如果你自行改了网络拦截配置，请确保解析器和页面模式对应正确。

通用配置项（两种模式共用）

配置项说明默认值
全局硬超时 单次请求的绝对超时上限，超过此时间无论如何都会结束。在现有情景下与首次超时并无绝对区别 300 秒
静默超时 (DOM) 网路监听模式下的静默超时阈值 3 秒
初始等待 等待 AI 开始回复的最长时间 30 秒

💡 关于超时配置：慢速思考模型（如 o1/长推理）建议提高「首次响应超时」与「全局硬超时」。两者不必完全相同：前者控制“首包等待”，后者控制“整轮上限”。

⚠️ 关于 arena.ai：该站点建议不要开启网络拦截模式。因为网络监听会创建额外的浏览器连接，增加被 Cloudflare 检测到的概率。请使用默认的 DOM 模式。

📄 文件粘贴

当发送的文本长度超过阈值时，系统会先创建一个临时 .txt 文件，再尝试把这个文件作为真实附件送进网页。它的目标不是单纯往输入框里塞超长文字，而是尽量走站点自己的上传入口，从而绕过部分网站的输入长度限制。

配置项说明

配置项默认值说明
启用站点预设决定是否开启文件粘贴模式
阈值50000 字符全局默认值；不同站点预设可以单独覆盖
引导文本完全专注于文件内容文件挂载成功后自动追加到输入框中的提示语

💡 新版工作顺序：系统会先尝试站点配置里的 file_input，再试通用 input[type=file]，必要时点击 upload_btn 后重试，或者向 drop_zone 投递文件；这些入口都失败后，才会回退到 Windows 剪贴板文件粘贴。

配置方式

在控制面板的「配置」选项卡中找到「文件粘贴」面板，可以针对每个站点（的每个预设）独立配置。真正需要一起看的不只是 enabled 和 threshold，还包括站点里的 upload_btn、file_input、drop_zone 这些上传入口选择器。

⚠️ 注意：系统现在会在上传后检查页面上是否真的出现了“文件已挂载”的信号。若文件并没有真正被页面接受，就会放弃文件粘贴并回退到普通文本输入。引导文本的作用，则是让模型把注意力放回文件内容本身。

当前默认预设里已开启的站点

  • aistudio.google.com
  • chatgpt.com
  • chat.deepseek.com
  • www.doubao.com
  • chat.qwen.ai

实战建议：如果你要给新站点开文件粘贴，不要只改一个开关。最稳的做法是先确认网页本身支持附件，再补齐 upload_btn / file_input / drop_zone 中至少一个可用入口，然后再测长文本发送。

🛡️ 隐身模式

隐身模式用于绕过网站的反自动化检测（如 Cloudflare 人机验证）。开启后，脚本的所有操作（点击、鼠标移动、滚动等）都会模拟真实人类行为。

何时需要开启

  • 需要开启：访问有 Cloudflare 防护的站点（如 arena.ai、chatgpt.com 等）
  • 不需要开启：访问无防护或低防护的站点（如 AI Studio、DeepSeek 等）

💡 如何开启：在控制面板中选择站点后，在页面顶部勾选「隐身模式」复选框即可。该设置是每个预设独立的。

隐身模式做了什么

行为普通模式隐身模式
鼠标点击直接 CDP 命令模拟真实鼠标按压（含 pressure、微移、释放偏移）
鼠标移动瞬间跳转人类化曲线移动
空闲时无动作随机微漂移（模拟手部抖动）
滚动直接调用纯 CDP 滚轮事件
操作间隔最小延迟随机化延迟（0.1-0.3 秒）

DrissionPage 补丁

为了最大程度减少被检测的风险，本项目需要对 DrissionPage 库进行一个小补丁，让网络监听功能复用浏览器的主连接而非创建额外的连接。

  • 自动执行：start.bat 启动脚本会在依赖安装后自动执行补丁
  • 手动执行：python patch_drissionpage.py
  • 恢复原文件：python patch_drissionpage.py --restore

⚠️ 注意：每次升级 DrissionPage 后需要重新执行补丁。补丁支持重复执行（幂等），不会重复打补丁。

⚠️ 关于 arena.ai 的特别说明：即便开启了隐身模式并关闭了网络监听，在半小时内连续对话约 10 条也几乎一定会触发 Cloudflare 人机验证。这是该站点本身的安全策略，真人操作也同样会触发，并非脚本问题。

⚡ 自动化命令

命令系统允许你设置触发条件和执行动作，实现标签页的自动化管理。例如：每对话 10 次自动清除 Cookie 并刷新页面，或检测到 Cloudflare 验证时自动切换代理节点。

📍 配置位置：进入 控制面板 → 命令

⚠️ 默认命令说明：项目默认内置了不少自动化命令，而且默认大多是禁用状态。这些命令主要是开发者针对 Gemini 和 Arena 场景编写的高度定制化恢复、探针和分流逻辑；如果你暂时不清楚某条命令的用途，建议先不要开启，避免误触发或影响正常使用。

两种模式

模式说明适用场景
简单模式 选择触发条件 + 配置动作列表，零代码操作 常规自动化需求
高级模式 直接编写 JavaScript 或 Python 脚本 复杂逻辑、自定义操作

触发条件（与代码一致）

类型说明示例
request_count 对话次数达到阈值（按作用域聚合计数） 每 10 次对话触发一次（all/domain/tab）
error_count 连续错误次数达到阈值 连续失败 3 次后触发
idle_timeout 标签页空闲超过指定时间 空闲 5 分钟后触发
page_check 页面出现指定内容 检测到 "Cloudflare" 时触发
command_triggered 当某条命令触发后，再触发本命令 A 成功后执行 B
command_result_match 匹配上游命令/步骤的返回值（等于/包含/不等于） JS 探针返回 CSS_FAILED 时触发
network_request_error 按 URL 规则 + 状态码拦截网络异常 命中 /queue/join + 429 时触发

💡 page_check 匹配规则：默认检查 document.body.innerText，仅在正文不可用时回退到 document.title。英文关键词（如 battle）按单词边界匹配，减少误命中；且为边沿触发，同一关键词持续存在时只触发一次，消失后再次出现才会再次触发。

🔀 支持与/或表达式（自 2.5.9 起）：在"检查文本"中使用 || 表示"或"（任意一个关键词出现即触发），使用 && 表示"且"（所有关键词同时出现才触发）。示例：出了点问题 || 发生错误（页面出现任一即触发）、Cloudflare && verify（两个同时出现才触发）。不支持混用 || 和 &&，混用时按 || 优先拆分。

作用范围

  • 所有标签页：命令对所有标签页生效
  • 指定域名：仅对包含特定域名的标签页生效（如 chatgpt.com）
  • 指定标签页：仅对特定编号的标签页生效（如 #1）

💡 request_count 计数规则：all 统计全部标签页请求总和；domain 统计匹配域名下所有标签页请求总和；tab 仅统计当前标签页。

命令优先级与基准点

  • 每条命令可设置 priority（任意整数，默认 2），数字越大优先级越高。
  • 系统请求基准优先级由 CMD_REQUEST_PRIORITY_BASELINE 控制（默认 2）。
  • 当命令优先级高于基准点时，命令会优先于新请求获取标签页执行。
  • 高优先级命令若包含 clear_cookies，会等待同域忙碌标签页结束后再执行，降低对并行会话干扰。

💡 环境变量示例：在 .env 中设置 CMD_REQUEST_PRIORITY_BASELINE=2 作为系统请求基准优先级，修改后需重启服务。

命令组（分组执行）

你可以把多条命令收纳到同一个命令组，便于批量执行和编排复用。

  • 在命令列表里可批量“收纳为命令组 / 解散分组 / 批量禁用命令”，也可对整组直接“执行 / 禁用 / 解散”。
  • 支持手动执行整组（优先在空闲标签页执行，返回成功/失败统计）。
  • 可在动作链中使用 execute_command_group 触发整组执行。
  • 组内命令按列表顺序执行，可选是否包含禁用命令。

条件分支：命令结果匹配

command_result_match 是命令系统的“大脑”。它会读取上游命令的执行结果（或某个具体步骤的结果），按规则判断是否触发下游命令。

字段含义建议
监听命令 要观察哪条命令 选择稳定输出结果的上游命令
目标步骤 可选；不填则匹配命令最终返回值 推荐绑定关键步骤（如 JS 探针步骤）
匹配规则 equals / contains / not_equals 状态码/状态字符串建议用 equals
期望值 需要命中的目标内容 如 CSS_FAILED / SUCCESS

外部告警：Webhook

send_webhook 可将异常信息推送到手机或群机器人，实现无人值守下的人类介入。

支持模板变量：{{tab_index}}、{{domain}}、{{network_status}}、{{network_url}}、{{timestamp}}。

{"msg":"标签页 #{{tab_index}} 在 {{domain}} 命中 {{network_status}}，URL={{network_url}}"}

网络异常拦截（自 2.5.6 起）

network_request_error 用于拦截底层 HTTP 异常（如 403/429/5xx）。

  • 支持关键词或正则匹配 URL。
  • 支持状态码白名单（如 403,429,500,502,503,504）。
  • 可勾选“命中后立即中断等待”，让恢复动作秒级执行。
  • 已独立于 stream 模式：即使站点是 DOM 模式，也会启动 event-only 网络监听用于拦截。

💡 正则输入位置：当规则类型切到“正则表达式”时，正则内容就在上方的 URL 输入框中填写。示例：.*/queue/join.*

🔀 代理切换功能

🧳 想沿用自己的浏览器登录态，应该怎么配？

如果你希望脚本启动后直接继承自己平时浏览器里的登录状态、Cookie 和扩展，请使用 独立复制出来的用户数据目录，不要直接把脚本指向你日常正在使用的系统默认目录。

⚠️ 重要：从 Chrome 136 开始，Chrome 对远程调试做了安全调整，默认 Chrome 数据目录不再接受 --remote-debugging-port。如果你把 BROWSER_PROFILE_DIR 直接写成系统默认的 User Data，浏览器可能会被打开，但调试端口不会真正起来，脚本也就无法接管它。Chrome 官方说明见：Changes to remote debugging switches to improve security。

推荐做法：复制一份专用用户目录
  1. 先完全关闭所有 Chrome 窗口。
  2. 新建一个专用目录，例如 C:\Users\你的用户名\AppData\Local\UniversalWebApiProfile。
  3. 从原始目录 C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data 中，至少复制 Local State 和你要使用的配置子目录（如 Default、Profile 1）到这个新目录。
  4. 在 .env 中把脚本指向这份副本，然后重新运行 start.bat。
BROWSER_PATH=C:\Program Files\Google\Chrome\Application\chrome.exe
BROWSER_PROFILE_DIR=C:\Users\你的用户名\AppData\Local\UniversalWebApiProfile
BROWSER_PROFILE_NAME=Default

字段含义：

  • BROWSER_PATH：浏览器程序路径。
  • BROWSER_PROFILE_DIR：用户数据根目录，应该指向 User Data 这一层，或者你复制出来的专用目录。
  • BROWSER_PROFILE_NAME：具体配置名称，常见值是 Default、Profile 1、Profile 2。

容易写错的地方：BROWSER_PROFILE_DIR 不要写成 ...\User Data\Default。正确写法是：

BROWSER_PROFILE_DIR=C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data
BROWSER_PROFILE_NAME=Default

也就是说，BROWSER_PROFILE_DIR 写“根目录”，BROWSER_PROFILE_NAME 写“具体配置名”。

如果你不在乎沿用自己的登录态：最省心的方式是留空 BROWSER_PROFILE_DIR，让项目继续使用根目录下的 chrome_profile/。这样隔离最彻底，也最不容易和日常 Chrome 互相影响。

  • 作用：它控制的是“更新时是否保留不变”，不是运行时启用/禁用开关。
  • 目录项：勾选 config/、app/、static/ 这类目录项时，会保留整个目录。

当前默认保留项：

  • config/sites.local.json：站点默认预设覆盖

  • venv/：Python 虚拟环境
  • logs/：日志目录
  • image/：图片输出目录
  • updater.py：更新脚本本体
  • .git/：Git 元数据目录
  • __pycache__/：Python 缓存目录
  • *.pyc：Python 编译缓存文件
  • backup_*/：更新前备份目录

❓ 常见问题

Q1: 为什么酒馆显示连接失败？

A: 因为酒馆点击「测试 API」时发送的请求信息过短。特别是当你同时使用酒馆测试 API 功能并且目标站点是 AI Studio 时，容易出现这个问题。建议直接发送对话消息进行测试，而非使用测试按钮。

Q2: 使用时频繁失败怎么办？

A: 请按以下几种情况进行排查：

  1. 人为干涉了工作流：在工作流执行时，你点击了脚本控制的浏览器页面的其他按钮，或者切换了页面。在脚本执行过程中，你不能操作脚本控制的浏览器，最多只能看着。
  2. 折叠边框：如果你手动把页面边框缩小，或折叠某些按钮，脚本可能无法操作元素。因此必须将折叠元素全部展开（浏览器可以最小化但不能缩小边框）。
  3. 网站状态异常：查看一下脚本控制的浏览器状态，可能触发了人机验证，可能单次发送的信息太长被网站拒绝，也可能发送的内容包含违规信息被拦截。
  4. 网络问题：网络波动或者代理切换了 IP，导致发送失败。这种情况可能引发脚本死锁，需要重启脚本或等待约 5 分钟。表现为：脚本后台显示发送成功，但浏览器中信息没有被实际发送或发送后又失败了。
  5. 不要在受控浏览器里乱开无关页：教程页现在不会再自动占用受控浏览器标签，但你仍然不应该手动在受控浏览器里打开搜索页、资讯页或其它普通网页。如果已经打开并出现异常，重启脚本即可。
  6. 站点配置失效：如果以上问题都排除了，可能是该网站的配置已过期需要更新。你可以在 GitHub 上提 Issue 或加群反馈。如果你有一定基础，也可以手动更新选择器配置。

Q3: 我同时打开了多个页面，脚本会怎么操作？

A: 脚本使用标签页池自动管理所有打开的标签页：

  • 默认路由（/v1/chat/completions）：自动选择一个空闲标签页处理请求
  • 指定站点域名（/url/gemini.com/v1/chat/completions）：自动匹配该站点的标签页
  • 指定标签页（/tab/1/v1/chat/completions）：使用编号为 1 的标签页
  • 你可以为不同标签页分配不同的预设，实现同一站点的多种用途（如对话 + 识图）
  • 当同时有多个请求时，空闲标签页会被依次分配；如果所有标签页都忙碌，新请求会排队等待

Q4: 标签页池中看不到新打开的标签页？

A: 请检查以下几点：

  • 新标签页的网页是否已经加载完成（`chrome://newtab` 等空白页不会被识别）
  • 等待 2-3 秒后点击「立即刷新」按钮
  • 如果仍然看不到，重启脚本即可

Q5: 控制面板的参数应该怎么调？需要调吗？

A: 大多数时候不需要调整，里面的参数都是配置好的默认值，可以满足绝大多数使用场景。

Q6: 有什么已知问题？

A: 目前已经有一些已知的问题：

  • VSCode 中已经发现 Codex 插件不适配，原因未知。
  • DrissionPage 补丁需在每次升级 DrissionPage 后重新执行。

Q7: 代理切换命令不生效怎么办？

A: 请检查以下几点：

  1. Clash 是否运行：确保 Clash 正在运行且 External Controller 已启用。
  2. API 地址是否正确：默认是 http://127.0.0.1:9090，部分 Clash 客户端可能使用其他端口。
  3. 代理组名称是否正确：访问 http://127.0.0.1:9090/proxies 查看实际的代理组名称。
  4. 是否设置了 Secret：如果 Clash 配置了 secret，需要在命令配置中填写。
  5. 查看日志：在控制面板的「日志」选项卡中查看 [CMD] 开头的日志，会显示切换结果或错误原因。

Q8: 我想直接使用自己原来的 Chrome 登录态，为什么配了还是不生效？

A: 最常见的原因有三个：

  • 直接指向了系统默认 User Data：在 Chrome 136+ 上，这通常会导致浏览器虽然打开了，但远程调试端口起不来。请改为复制出一份独立目录，再把脚本指向那份副本。
  • BROWSER_PROFILE_DIR 写错层级：它应该写到 User Data 这一层，而不是 ...\User Data\Default。具体配置名请写在 BROWSER_PROFILE_NAME。
  • 配置改完没重启启动脚本：这类浏览器启动配置必须重新运行 start.bat 才会生效。

详细步骤请回看上方「环境配置 → 想沿用自己的浏览器登录态，应该怎么配？」这一节。

Q9: 这个项目有什么用？

A: 这是一个「白嫖」项目，主要用途包括：

  • 免费额度利用：很多 AI 厂商在网页端有相当多的免费额度（如 DeepSeek、AI Studio、Arena AI 等），本项目能将网页额度转化为 OpenAI 格式的 API，供前端调用。
  • 调试上下文：你可以便捷地查看前端是如何构造上下文的——只需打开浏览器复制一遍脚本发送的请求内容即可。
  • 多标签页 + 多预设：同时开多个标签页，为不同标签页配置不同预设（如主端口用于聊天，副端口用于识图），实现多用途并行。
  • 长文本绕过：通过文件粘贴功能，绕过部分网站输入框的字符限制。
